StockBook follows Clean Architecture principles with clear separation of concerns:

  • Domain Layer: Core business logic, entities, and value objects
  • Application Layer: Use cases, commands, and application services
  • Infrastructure Layer: Database persistence, external services
  • Presentation Layer: REST API endpoints using FastAPI

üîß Generating Diagrams

To regenerate the architecture diagrams from the current codebase:

# Generate diagrams (PlantUML source files)
make docs-arch

# Generate and serve diagrams locally
make docs-serve

To render diagrams as images, install PlantUML:

# Ubuntu/Debian
sudo apt-get install plantuml

# macOS
brew install plantuml

# Then generate PNG/SVG files
cd docs/architecture/diagrams
plantuml -tpng *.puml
plantuml -tsvg *.puml
